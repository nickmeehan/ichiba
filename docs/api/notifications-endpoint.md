# Notifications Endpoint

The Notifications API manages user notifications including delivery, read status, preferences, and push device registration. Notifications are generated by system events, mentions, and automations.

## Endpoints

| Method | Path | Description |
|--------|------|-------------|
| GET | `/v1/notifications` | List notifications for the authenticated user |
| GET | `/v1/notifications/{id}` | Get notification details |
| POST | `/v1/notifications/{id}/read` | Mark a notification as read |
| POST | `/v1/notifications/read-all` | Mark all notifications as read |
| POST | `/v1/notifications/{id}/unread` | Mark a notification as unread |
| GET | `/v1/notifications/preferences` | Get notification preferences |
| PATCH | `/v1/notifications/preferences` | Update notification preferences |
| POST | `/v1/notifications/push/register` | Register a push device |
| DELETE | `/v1/notifications/push/{device_id}` | Unregister a push device |

## Listing Notifications

```
GET /v1/notifications?filter[read]=false&sort=-created_at
```

```json
{
  "data": [
    {
      "id": "notif_001",
      "type": "mention",
      "title": "Jane Chen mentioned you in a comment",
      "body": "...can you review the login flow?",
      "read": false,
      "resource": { "type": "comment", "id": "comment_101", "task_id": "task_42" },
      "actor": { "id": "user_05", "display_name": "Jane Chen" },
      "created_at": "2026-03-15T10:30:00Z"
    }
  ],
  "meta": {
    "unread_count": 12
  }
}
```

## Notification Types

| Type | Trigger |
|------|---------|
| `mention` | User is @mentioned in a comment |
| `assignment` | Task assigned to user |
| `status_change` | Task the user watches changes status |
| `comment` | New comment on a watched task |
| `due_date` | Task due date approaching or passed |
| `sprint` | Sprint started, completed, or updated |
| `milestone` | Milestone due date approaching |
| `reaction` | Someone reacted to user's comment |
| `permission` | Permission granted or revoked |
| `system` | System announcements and maintenance |

## Mark Read/Unread

```
POST /v1/notifications/notif_001/read
POST /v1/notifications/read-all
```

`read-all` accepts optional filters to mark a subset as read:

```json
POST /v1/notifications/read-all
{
  "filter": { "type": "comment", "resource.task_id": "task_42" }
}
```

## Notification Preferences

Configure which notification types are delivered via which channels:

```json
PATCH /v1/notifications/preferences
{
  "mention": { "in_app": true, "email": true, "push": true },
  "assignment": { "in_app": true, "email": true, "push": false },
  "comment": { "in_app": true, "email": false, "push": false },
  "due_date": { "in_app": true, "email": true, "push": true }
}
```

## Push Registration

Register a device for push notifications:

```json
POST /v1/notifications/push/register
{
  "platform": "ios",
  "token": "device_push_token_abc123",
  "device_name": "iPhone 15"
}
```

Platforms: `ios`, `android`, `web`.
