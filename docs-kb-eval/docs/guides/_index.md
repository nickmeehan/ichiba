# Guides

Practical how-to guides for common tasks and workflows in the Nimbus platform. Each guide walks you through a specific scenario from start to finish.

## Available Guides

| Guide | When to use it |
|-------|---------------|
| [Onboarding](onboarding.md) | You just joined the team and need to get your development environment running, understand the codebase, and ship your first PR. |
| [Debugging](debugging.md) | Something is broken and you need to trace the problem through the stack, from React components down to database queries. |
| [Local Development](local-dev.md) | You need to set up, configure, or troubleshoot your local Nimbus development environment with Docker Compose. |
| [Code Review](code-review.md) | You are reviewing a pull request or preparing your own PR for review and want to follow team standards. |
| [Feature Flags](feature-flags.md) | You need to gate a feature behind a flag, configure a gradual rollout, or clean up stale flags after a launch. |
| [Performance Profiling](performance-profiling.md) | A page or API endpoint is slow and you need to identify the bottleneck using profiling and tracing tools. |
| [Data Migration](data-migration.md) | You need to transform, move, or backfill data across tenants without downtime or data loss. |
| [Incident Response](incident-response.md) | A production incident is happening and you need to know who to contact, how to communicate, and what to do next. |
| [Deployment](deployment/) | You need to deploy code to staging or production, set up canary releases, or roll back a bad deploy. |
| [Security](security/) | You need guidance on threat modeling, input validation, secrets management, or compliance requirements. |
