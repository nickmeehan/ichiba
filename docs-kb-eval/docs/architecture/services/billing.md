# Billing Service

The billing service manages subscription plans, payment processing, and usage tracking for Nimbus workspaces. All payment processing is handled through Stripe, and the service is designed to be the single source of truth for a workspace's entitlements.

## Subscription Tiers

| Tier       | Price         | Limits                                          |
|------------|---------------|--------------------------------------------------|
| Free       | $0/month      | 5 members, 3 projects, 100 MB storage            |
| Pro        | $12/user/month| Unlimited members, unlimited projects, 10 GB storage |
| Enterprise | Custom        | SSO, SCIM, audit log export, dedicated support    |

## Stripe Integration

Each workspace has a corresponding Stripe Customer object. Subscriptions are managed through the Stripe Billing API with the following flow:

1. User selects a plan in the Nimbus billing settings.
2. Nimbus creates a Stripe Checkout Session.
3. User completes payment on the Stripe-hosted checkout page.
4. Stripe sends a `checkout.session.completed` webhook.
5. Nimbus activates the subscription and updates entitlements.

## Usage-Based Add-Ons

Some features are billed based on usage beyond the plan limits:

| Add-On          | Unit           | Price              |
|-----------------|----------------|--------------------|
| Extra Storage   | Per GB/month   | $0.10              |
| API Calls       | Per 10K calls  | $0.50              |
| Automation Runs | Per 1K runs    | $1.00              |

Usage is metered hourly and reported to Stripe using metered billing items. Usage records are stored locally in ClickHouse for dashboard display.

## Webhook Handling

The billing service listens for Stripe webhooks on `/api/webhooks/stripe`. Events are verified using the webhook signing secret before processing. Key events handled:

| Stripe Event                     | Action                                       |
|---------------------------------|----------------------------------------------|
| `invoice.payment_succeeded`      | Record payment, extend subscription           |
| `invoice.payment_failed`         | Notify workspace owner, start grace period    |
| `customer.subscription.deleted`  | Downgrade to Free tier                        |
| `customer.subscription.updated`  | Update local entitlements                     |

## Invoice Generation

Monthly invoices are generated by Stripe and accessible through the Nimbus billing dashboard. Workspace owners can download PDF invoices and update billing details (company name, VAT ID, billing address) which sync to the Stripe Customer object.

## Entitlement Checks

Feature gating is performed through an `EntitlementService` that checks the workspace's active subscription tier. This is used by middleware to enforce limits:

```typescript
await entitlements.require('feature.sso', tenantId);
// Throws UpgradeRequiredError if not on Enterprise plan
```
